# nix-config

Personal Nix configuration for macOS and NixOS machines, managed with
[flakes](https://nixos.wiki/wiki/Flakes) and
[home-manager](https://github.com/nix-community/home-manager).

## Structure

This repo follows the **[dendritic pattern](https://github.com/mightyiam/dendritic)**:
every `.nix` file (except entry points) is a self-contained top-level
module, auto-discovered via [import-tree](https://github.com/vic/import-tree).
No manual `imports` lists.

```
flake.nix                      flake-parts entry point
modules/
  options.nix                   dotfiles.* option declarations + config defaults
  home.nix                      home-manager base (fonts, nix.conf, darwin helpers)
  flake/
    configurations.nix          mkHome/mkNixos — builds homeConfigurations & nixosConfigurations
    checks.nix                   eval checks (home + nixos)
    formatter.nix               nix fmt
  features/                     cross-platform home-manager modules
    ai/                          AI coding tool config (oh-my-pi, maki, providers, smolvm)
    editor.nix                  helix + LSP servers
    shell.nix                   fish + starship + zellij integration
    packages.nix                system packages
    git.nix                     git config + hunk diff wrapper
    theme.nix                   base16 theming (dark/light switching)
    zellij.nix                  zellij config
    terminal.nix, ssh.nix, file-manager.nix, calibre.nix
    repos/                      ghq repo management scripts
  desktop/                      desktop-environment modules
    niri/                       niri compositor (keybinds, settings, noctalia integration)
    aerospace.nix               macOS aerospace tiling WM config
    linux-apps.nix             Linux desktop services
  nixos/                        NixOS system modules (base, networking, sound, ssh, etc.)
  hosts/                        per-host hardware config (generated by nixos-generate-config)
  lib/                          helper libraries
  wallpapers/                   LFS-tracked wallpaper images
  tests/                        niri equalize test
bootstrap.sh                    idempotent machine setup (clone, home-manager switch, SSH keys, gh auth)
```

## Installation

### Fresh machine (requires only Nix)

```sh
curl -fsSL bootstrap.sammohr.dev | bash
```

This clones the repo via `ghq`, runs `home-manager switch`, generates SSH
keys, authenticates GitHub, and (on NixOS) applies the system configuration.

### Manual

```sh
nix run github:nix-community/home-manager -- switch --flake .#smores@smortress --no-write-lock-file
```

On NixOS, apply the system config after home-manager:

```sh
sudo nixos-rebuild switch --flake .#smortress
```

### Adding a new host

1. Run `nixos-generate-config` on the target hardware, move the generated
   `configuration.nix` to `modules/hosts/<hostname>.nix`, and strip to
   hardware-only settings (filesystems, kernel modules, CPU microcode).
2. Add a `nixosConfigurations."<hostname>"` entry in
   `modules/flake/configurations.nix` via `mkNixos`.
3. Add a `homeConfigurations` entry via `mkHome`.

## Dendritic pattern in this repo

Each `.nix` file under `modules/` is auto-imported by `import-tree` (called
in `modules/flake/configurations.nix`). A file under `modules/nixos/` is
a NixOS module; under `modules/features/` or `modules/desktop/` is a
home-manager module; under `modules/flake/` is a flake-parts module.

Cross-cutting values flow through `config.dotfiles.*` options declared in
`modules/options.nix`, not through `specialArgs` pass-through. Every
module reads what it needs from `config.dotfiles`, and each host config
sets its own options in `mkHome`/`mkNixos` calls.
