NixOS system configs and home-manager user configs for all my machines.

## Hosts

| Host | System | Desktop | Config type |
|---|---|---|---|
| `smohr` | macOS (aarch64) | Aerospace | home-manager |
| `smoresbook` | NixOS | Niri + Noctalia | NixOS + home-manager |
| `smorestux` | NixOS | Niri + Noctalia | NixOS + home-manager |

## Usage

```bash
# Home-manager
home-manager switch --flake ~/.config/nix#smores@<hostname>   # Linux
home-manager switch --flake ~/.config/nix#smohr               # macOS

# NixOS
sudo nixos-rebuild switch --flake ~/.config/nix#<hostname> --upgrade

# Format all nix files
nix fmt
```

## New machine setup

```bash
nix-shell -p git gh home-manager helix
ssh-keygen -t ed25519 -b 4096 -C <hostname>
gh auth login
gh repo clone smores56/nix-config ~/.config/nix
home-manager switch --flake ~/.config/nix#$USER@$(hostname)

# NixOS only:
sudo nixos-generate-config
cp /etc/nixos/hardware-configuration.nix ~/.config/nix/hosts/$(hostname)/hardware.nix
sudo nixos-rebuild switch --flake ~/.config/nix#$(hostname) --upgrade
```

## Architecture

`flake.nix` defines `mkHome` and `mkNixos` builders. Each host is a small attrset of properties (`displayManager`, `helixTheme`, etc.). Per-host values are validated by `modules/options.nix` and `modules/nixos/options.nix` using `lib.mkOption` with `lib.types.enum` ‚Äî invalid values produce clear errors at eval time.

Display-manager-specific modules are dispatched via **attrset lookup** in three places ‚Äî adding a new display manager is one entry in each:

- `flake.nix` ‚Üí `flakeModules` (external flake modules like niri/noctalia)
- `modules/host.nix` ‚Üí `displayManagerModules` (NixOS modules)
- `modules/gui/default.nix` ‚Üí `displayManagerModules` (home-manager GUI modules)

Consuming modules access per-host config through `config.dotfiles.*` (typed options) and `pkgs.stdenv.isLinux` for platform detection. The only remaining `specialArg` is `displayManager` (needed for imports-level dispatch, which can't depend on `config`).

### Module structure

- `modules/home.nix` ‚Äî shared home-manager base, imports `tools/` and `gui/`
- `modules/host.nix` ‚Äî shared NixOS base, dispatches to DM-specific NixOS modules
- `modules/gui/` ‚Äî DM-specific GUI modules + `terminal-emulator/` (4 terminals, config-only via `package = pkgs.nil`)
- `modules/tools/` ‚Äî CLI tools, shell, editor, theming (all hosts)
- `modules/nixos/` ‚Äî shared NixOS services (networking, bluetooth, sound, etc.)
- `hosts/*/hardware.nix` ‚Äî per-host hardware configs (auto-generated by NixOS)
